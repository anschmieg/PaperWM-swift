# Contributing to PaperWM-swift

Thank you for your interest in contributing to PaperWM-swift! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/PaperWM-swift.git`
3. Add the upstream remote: `git remote add upstream https://github.com/anschmieg/PaperWM-swift.git`
4. Create a branch for your changes: `git checkout -b feature/your-feature-name`

## Development Setup

1. Ensure you have Swift 5.9 or later installed
2. Build the project: `swift build`
3. Run tests: `swift test`

## Making Changes

### Code Style

- Follow Swift naming conventions
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and small

### Testing

- Add tests for new features
- Ensure all tests pass before submitting a PR
- Run `swift test` to verify

### Commits

- Write clear, descriptive commit messages
- Use present tense ("Add feature" not "Added feature")
- Reference issues in commit messages when applicable

## Submitting Changes

1. Push your changes to your fork
2. Create a Pull Request against the main repository
3. Describe your changes in the PR description
4. Link to any related issues
5. Wait for review and address any feedback

## Pull Request Guidelines

- Keep PRs focused on a single feature or fix
- Update documentation as needed
- Add tests for new functionality
- Ensure all tests pass
- Keep commit history clean and meaningful

## Code Review Process

1. A maintainer will review your PR
2. Address any requested changes
3. Once approved, your PR will be merged

## Reporting Bugs

When reporting bugs, please include:

- macOS version
- Swift version
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Any error messages or logs

## Feature Requests

Feature requests are welcome! Please:

- Check if the feature has already been requested
- Describe the feature and its use case
- Explain why it would be valuable

## Questions?

If you have questions, please:

- Check the documentation (README.md, INTEGRATION.md, QUICKSTART.md)
- Search existing issues
- Create a new issue with the "question" label

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers
- Accept constructive criticism
- Focus on what's best for the project
